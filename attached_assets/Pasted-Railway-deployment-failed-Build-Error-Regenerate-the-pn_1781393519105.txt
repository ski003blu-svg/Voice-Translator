## Problem
The Voice-Translator service is a pnpm workspace (9 workspace projects) built with Railpack. The build fails at the `pnpm install --frozen-lockfile --prefer-offline` step with:

```
ERR_PNPM_LOCKFILE_CONFIG_MISMATCH  Cannot proceed with the frozen installation. The current "overrides" configuration doesn't match the value found in the lockfile

Update your lockfile using "pnpm install --no-frozen-lockfile"
```

The lockfile is at `pnpm-lock.yaml` (sha: 21efe768b1ff3900a4bc3c093c315d4dbca2dda2). The workspace config is in `pnpm-workspace.yaml`.

## Diagnosis
The `pnpm-workspace.yaml` file contains an extensive `overrides` section that excludes platform-specific optional packages for esbuild, lightningcss, @tailwindcss/oxide, rollup, and @expo/ngrok-bin, and also pins `esbuild` to version `0.27.3`. These overrides are not reflected in the committed `pnpm-lock.yaml`. Railway runs `pnpm install --frozen-lockfile`, which aborts when it detects any mismatch between the current workspace configuration and the lockfile. The triggering commit only changed `artifacts/api-server/src/routes/voice.ts` (a code refactor), so the lockfile was already out of sync before this deployment. No commits have been pushed to main after the failed deployment.

## Fix
This fix cannot be performed by a file-editing agent alone because it requires running `pnpm install` to regenerate the lockfile binary/text output.

1. Check out the `main` branch locally.
2. Run `pnpm install` (without `--frozen-lockfile`) from the repo root. This will update `pnpm-lock.yaml` to reflect the current `overrides` in `pnpm-workspace.yaml`.
3. Commit the updated `pnpm-lock.yaml`:
   ```
   git add pnpm-lock.yaml
   git commit -m "chore: regenerate lockfile to match pnpm-workspace.yaml overrides"
   git push origin main
   ```
4. Railway will automatically trigger a new deployment, which will now pass the `pnpm install --frozen-lockfile` step.
